glassfish:create-domain

Create a new domain in a local or remote Glassfish instance

Mojo Attributes :

  • Requires a Maven 2.0 project to execute.
  • Automatically executes within the lifecycle phase: pre-integration-test
  • Requires that mvn runs in online mode.

Required Parameters

Name Type Description
domain Domain Container for domain configuration parameters.
glassfishDirectory File The root directory of the Glassfish installation to be used

Optional Parameters

Name Type Description
domainDirectory File The directory into which domains are deployed Default value is ${glassfishDirectory}/domains.
debug boolean Debug Glassfish output Default value is false.
echo boolean Echo Glassfish asadmin commands Default value is false.
terse boolean Terse Glassfish output Default value is true.
skip boolean Skip execution Default value is false.
autoCreate boolean Automatically create the domain if it does not already exist Default value is true.
user String The unprivileged user to run as
passwordFile String Location of the asadmin style password file (if you do not want to provide the password in your POM)
adminPassword String The admin password to use for this domain (if you would rather not use an asadmin style password file)

Parameter Details

domainDirectory

The directory into which domains are deployed

  • Type : java.io.File
  • Required : No
  • Default : ${glassfishDirectory}/domains

domain

Container for domain configuration parameters.

  • Type : org.glassfish.maven.plugin.Domain
  • Required : Yes

glassfishDirectory

The root directory of the Glassfish installation to be used

  • Type : java.io.File
  • Required : Yes
  • Expression : ${glassfish.home}

debug

Debug Glassfish output

  • Type : boolean
  • Required : No
  • Default : false

echo

Echo Glassfish asadmin commands

  • Type : boolean
  • Required : No
  • Default : false

terse

Terse Glassfish output

  • Type : boolean
  • Required : No
  • Default : true

skip

Skip execution

  • Type : boolean
  • Required : No
  • Default : false

autoCreate

Automatically create the domain if it does not already exist

  • Type : boolean
  • Required : No
  • Default : true

user

The unprivileged user to run as

  • Type : java.lang.String
  • Required : No
  • Expression : ${user.name}

passwordFile

Location of the asadmin style password file (if you do not want to provide the password in your POM)

  • Type : java.lang.String
  • Required : No

adminPassword

The admin password to use for this domain (if you would rather not use an asadmin style password file)

  • Type : java.lang.String
  • Required : No
